# 🎵 Music Match - 나만을 위한 스마트 음악 추천 웹사이트

> **대학교 웹 프로그래밍 과제 제출용 프로젝트**  
> 본 웹사이트는 Spotify와 Apple Music의 현대적인 UI/UX를 벤치마킹하여 제작된 반응형 음악 추천 서비스 포트폴리오입니다.

---

## 📂 프로젝트 파일 구성
본 프로젝트는 추가적인 빌드 도구나 프레임워크(React, Vue 등) 없이 브라우저 상에서 즉시 실행이 가능하도록 순수 **HTML5, CSS3, Vanilla JavaScript**로만 제작되어 평가 및 채점이 매우 용이합니다.

```text
music-match/
├── index.html        # 메인 구조 및 SPA(Single Page) 탭 세그먼트 구성 파일
├── styles.css        # Spotify 테마의 다크 모드, 네온 그린 포인트 및 반응형 그리드 스타일시트
├── app.js            # 30곡의 고품질 샘플 음악 데이터셋 및 동적 렌더링, 검색, 필터링, 평점, 즐겨찾기 스크립트
└── README.md         # 프로젝트 안내 및 설치/실행 가이드 (본 파일)
```

---

## ✨ 핵심 구현 기능

1. **메인 페이지 (홈)**
   - **오늘의 추천곡**: 사이트 접속 시마다 30곡 중 랜덤으로 1곡을 멋지게 큐레이션하여 히어로 배너로 제공합니다.
   - **TOP 10 인기곡**: 30곡 중 추천 수(좋아요) 기준 상위 10곡을 가로 슬라이더 형태로 순위(Rank 1~10)와 함께 노출합니다.
   - **최신 노래 추천**: 최근 발매일 기준 6곡을 배치하여 사용자가 최신 음악 트렌드를 파악할 수 있도록 돕습니다.

2. **장르별 추천 및 필터링**
   - 발라드, 힙합, 팝, 록, K-POP, 클래식의 6개 핵심 장르 필터를 완벽히 제공합니다.
   - 각 필터 버튼 클릭 시 해당 장르의 곡들만 실시간으로 그리드 레이아웃에 표시됩니다.

3. **분위기별 추천 및 필터링**
   - 공부할 때, 운동할 때, 드라이브할 때, 잠들기 전, 기분이 우울할 때, 기분이 좋을 때 등 실시간 감정/상태별 어울리는 곡 목록을 추출합니다.

4. **검색 기능**
   - 상단 실시간 검색창을 통해 **곡 제목** 또는 **가수 이름**의 일부분만 타이핑해도 즉시 필터링된 결과가 실시간 렌더링됩니다.

5. **즐겨찾기 (보관함) 기능**
   - 카드 우측 상단의 하트 또는 상세 정보 팝업 내 버튼을 누르면 나만의 음악으로 추가됩니다.
   - 브라우저를 종료하거나 새로고침해도 즐겨찾기 내역이 유지되도록 `localStorage`와 연동되어 있습니다.

6. **노래 상세 정보 및 1~5점 별점 평가**
   - 음악 카드를 클릭하면 상세 정보 모달이 뜨며, 고화질 앨범 아트와 함께 **추천 이유**를 자세히 확인할 수 있습니다.
   - 모달 하단에서 사용자가 직접 별점을 클릭하여 점수(1.0 ~ 5.0)를 입력할 수 있으며, 이 역시 즉각 카드 평점에 반영되고 브라우저에 저장됩니다.

7. **완벽한 반응형 웹 지원**
   - 미디어 쿼리를 사용해 **데스크톱(3단 레이아웃)**, **태블릿(2단)**, **모바일(1단 + 슬라이드인 햄버거 메뉴)** 환경의 가로세로 비율에 최적화된 유연한 반응형 화면을 제공합니다.

## 🚀 실행 및 테스트 방법

### 방법 1. 로컬 환경에서 바로 실행 (더블 클릭)
1. 프로젝트 폴더 내 `index.html` 파일을 크롬(Chrome)이나 엣지(Edge), 사파리(Safari) 브라우저로 마우스 더블 클릭하여 바로 실행할 수 있습니다.
2. 로컬 웹 서버가 필요 없이 간편하게 작동합니다.

### 방법 2. VS Code의 Live Server 활용 (권장)
1. Visual Studio Code에서 프로젝트 폴더를 엽니다.
2. 우측 하단의 `Go Live` 버튼을 클릭하거나, `index.html` 파일 우클릭 후 `Open with Live Server`를 클릭합니다.
3. `http://127.0.0.1:5500` 포트에서 실행되며, 코드를 수정할 때 브라우저가 자동 리로드(Live Reloading)됩니다.

## 💡 과제 평가 포인트 (A+ 공략 요소)
- **주석의 체계성**: `app.js` 및 `styles.css`에 각 구문별/기능별 상세한 주석을 달아 채점관(교수님 또는 조교님)이 코드를 쉽게 이해할 수 있도록 구조화했습니다.
- **예외 처리 & UX 디테일**: 검색 결과가 없을 때의 Empty State 페이지 디자인, 랜덤 룰렛 애니메이션 스피너 연출, 반응형 스크롤바 튜닝 등 상용 서비스 수준의 완성도를 확보했습니다.
- **데이터 영속성**: `localStorage`를 사용해 즐겨찾기 상태와 개별 별점을 로컬 디스크에 보존하는 기술적 디테일을 적용했습니다.
